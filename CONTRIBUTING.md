# Contribution Guide | 贡献指南

## English

Thank you for your interest in contributing to Operator Lambda! This project is open to code contributions, documentation improvements, bug reports, and feature suggestions.

### How to contribute

1. **Report bugs or suggest features** by opening an [Issue](https://github.com/hdkghc/calculator/issues). Please include enough context to reproduce the problem or understand the proposal.

2. **Submit code changes** via Pull Request:
   - Fork the repository and create a new branch for your changes.
   - Keep the code style consistent with the existing codebase.
   - Write clear, concise commit messages.
   - If your change affects behavior, update the documentation accordingly.

3. **Test your changes**: Run the existing tests on your computer before submitting. Ensure that your changes do not break existing functionality.

### Pull Request guidelines

- PRs should target the `main` branch.
- Keep PRs focused on a single topic. Split large changes into multiple smaller PRs if possible.
- Link related issues in the PR description.
- The PR will be reviewed; you may be asked to make revisions.

By contributing, you agree that your contributions will be distributed under the same license as the project: **GPLv3**. See [LICENSE](LICENSE) for details.

## 中文

感谢你对 Operator Lambda 的关注与贡献！本项目欢迎代码、文档、Bug 报告和功能建议。

### 如何参与

1. **报告 Bug 或提出建议**：请在 [Issues](https://github.com/hdkghc/calculator/issues) 中提出，并附上足够的上下文信息，以便复现问题或理解你的想法。

2. **提交代码**：请通过 Pull Request 提交：
   - Fork 本仓库，为你的更改创建独立分支。
   - 保持代码风格与现有代码一致。
   - 提交信息应清晰、简洁。
   - 如果更改影响行为，请同时更新相关文档。

3. **测试你的更改**：在电脑上运行现有测试，确保你的更改没有破坏现有功能。

### Pull Request 规范

- PR 目标分支应为 `main`。
- 一个 PR 聚焦单一主题。较大的改动请拆分为多个较小的 PR。
- 在 PR 描述中关联相关 Issue。
- PR 会经过审查，可能需要进行修改。

### 许可证

贡献即表示你同意你的贡献将以项目相同的许可证 **GPLv3** 发布。详见 [LICENSE](LICENSE) 文件。
