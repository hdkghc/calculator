# Contribution Guide | 贡献指南

## English
Thank you for your interest in contributing to this project! We welcome code, documentation, and ideas. Please follow these guidelines:

1. **Fork the repository** and create your feature branch.
2. **Write clear, well-documented code**. Add comments where necessary.
3. **Test your changes** before submitting a pull request.
4. **Describe your changes** clearly in the pull request.
5. For major features or design changes, please open an issue first to discuss.
6. By contributing, you agree to license your work under GPLv3.

## 中文
感谢你对本项目的关注与贡献！我们欢迎代码、文档和创意。请遵循以下指南：

1. **Fork 本仓库**，创建你的功能分支。
2. **编写清晰、文档完善的代码**，必要时添加注释。
3. **在提交前测试你的更改**。
4. **在 Pull Request 中清楚描述你的更改内容**。
5. 对于重大特性或设计变更，请先提交 issue 讨论。
6. 贡献即视为同意以 GPLv3 许可协议发布你的工作。
